# Model Card: `audiobook-ai-tn-byt5-small`

A sequence-to-sequence **Text Normalization (TN)** model that converts *written* text into its *spoken* form for a Text-to-Speech (TTS) front-end. It is the trainable ML core of the **Audiobook Generation System** (P05, NLP in Industry final assignment), which turns long documents (EPUB/PDF/TXT/MD) into mastered, chaptered audiobooks.

- **Author:** Le Dinh Minh Quan (student 23127460)
- **Base model:** [`google/byt5-small`](https://huggingface.co/google/byt5-small) (Apache-2.0)
- **Architecture:** Seq2Seq (encoder-decoder), byte/char-level
- **Task prefix:** `normalize: `
- **Language:** English
- **License:** Apache-2.0 (inherited from base)

---

## 1. Model Details

| Field | Value |
|---|---|
| Model name | `audiobook-ai-tn-byt5-small` |
| Base | `google/byt5-small` (~300M params, Apache-2.0, byte/char-level) |
| Small-GPU fallback base | `google-t5/t5-small` (60.5M params, Apache-2.0) |
| Type | Seq2Seq text-to-text |
| Input | Written sentence prefixed with `normalize: ` |
| Output | Spoken-form rendering of the sentence |
| Tokenization | Byte-level (ByT5) — no subword vocabulary, operates on raw UTF-8 bytes |

ByT5 is byte/char-level, which makes it **robust to numbers, symbols, and out-of-vocabulary tokens** — exactly the inputs that break naive TTS front-ends. The model normalizes text across **16 semiotic classes**:

`PLAIN`, `PUNCT`, `CARDINAL`, `ORDINAL`, `DECIMAL`, `DIGIT`, `MONEY`, `MEASURE`, `DATE`, `TIME`, `FRACTION`, `TELEPHONE`, `ELECTRONIC`, `ABBREV`, `ROMAN`, `VERBATIM`.

### Why this model exists

A naive TTS engine reads `"$5.2M"`, `"1984"`, `"Dr."`, `"Chapter IV"`, `"3/4"`, and `"9:45 AM"` incorrectly. The hard, NLP-heavy part of audiobook production is **text normalization (written → spoken)**. This model is the *trainable* heart of the pipeline; audio is produced by a separate pretrained neural TTS backend, and an agent orchestrates the full document → audiobook flow.

---

## 2. Intended Use

### Direct / primary use
- **TTS front-end normalization** for English audiobook production.
- Per-segment normalization inside the Audiobook Generation pipeline, called after sentence segmentation and before speech synthesis.
- Producing spoken forms for numbers, money, dates, times, fractions, measures, abbreviations, Roman numerals, telephone numbers, and electronic addresses **using sentence context** (e.g., disambiguating `St.` → *Street* vs *Saint*, or `1984` as a count vs a year).

### Intended users
- Engineers building TTS / audiobook / accessibility systems (sight-impaired, dyslexic, and commuter listeners) who need a context-aware normalization stage in front of a neural vocoder.

### Out-of-scope use
- **Other languages.** The model is trained on English only; non-English input is out of scope.
- **Inverse normalization / ASR direction (ITN).** This model performs written → spoken normalization only. It is not designed for spoken → written conversion (e.g., ASR post-processing).
- **Non-TTS downstream tasks** (e.g., as a general text rewriter, translator, or summarizer).
- Any use that assumes the output is the *original* written text — the output is a deliberately *spoken* rendering.

## 3. Training Data

There is **no permissively-licensed English TN-Challenge mirror** available on the HF Hub (the canonical ids `google/text_normalization` and `cestwc/text-normalization` return 404). The model is therefore trained on a **reproducible synthetic TN corpus**.

### Synthetic corpus
- Generated by `src/audiobook_ai/data/tn_corpus.py`, producing **context-rich `(written, spoken)` pairs** across all 16 semiotic classes.
- **Injected ambiguity** so the model must learn context, not surface rules. Examples:
  - `St.` → *Street* vs *Saint* depending on context.
  - `"1984 people"` → cardinal count vs `"in 1984"` → year.
  - Dates with day-as-ordinal.
  - Roman-numeral contexts that a context-blind baseline misses.

### Default split sizes

| Split | Size |
|---|---|
| Train | 60,000 |
| Validation | 4,000 |
| Test | 4,000 |
| Hard slice | 1,500 |

- **Leakage-free splits:** no validation/test input appears in the training set.
- **Optional real-data loader** is available, plus an optional eval sanity set: [`DigitalUmuganda/Text_Normalization_Challenge_Unittests_Eng_Fra`](https://huggingface.co/datasets/DigitalUmuganda/Text_Normalization_Challenge_Unittests_Eng_Fra).

### License
The synthetic corpus is generated by the project's own code and the model inherits the **Apache-2.0** license of its base. The optional real eval set is used under its own upstream terms.

## 4. Training Procedure

Training uses the Hugging Face `Seq2SeqTrainer` with `predict_with_generate=True`.

### Hyperparameters

| Hyperparameter | Value |
|---|---|
| Learning rate | `5e-4` (ByT5) / `3e-4` (t5-small fallback) |
| Effective batch size | 256 (e.g., per-device 32 × grad-accum 8 on H100) |
| LR schedule | Cosine |
| Warmup ratio | 0.05 |
| Weight decay | 0.01 |
| Label smoothing | 0.1 |
| Dropout | 0.1 |
| Max grad norm | 1.0 |
| Precision | bf16 + tf32 (H100/A100) |
| Early stopping | Patience 4 on `sentence_accuracy` |
| Checkpointing | `load_best_model_at_end`; resume via `get_last_checkpoint` |

### Hardware profiles (effective batch held at 256)

| GPU | Precision | Per-device batch | Grad-accum | Grad-checkpointing | Notes |
|---|---|---|---|---|---|
| H100 | bf16 + tf32 | 32 | 8 | off | Reference profile |
| A100-40 | bf16 | 16 | 16 | on | |
| L4 | bf16 | 8 | 32 | on | |
| T4 | fp16 | 4 | 64 | on | Switch base to `t5-small` |

### Approximate training time
- **ByT5-small:** ~3–6 h on a single H100 for the full corpus (or cap `max_steps`).
- **t5-small:** ~3–4× faster; fits on a T4.

### Anti-overfitting measures
- Deduplicate `(written, spoken)` pairs **before** splitting.
- Cap `PLAIN`-class share at ~30% so easy cases don't dominate.
- Early stopping on the validation sentence-accuracy.
- Label smoothing (0.1), weight decay (0.01).
- A held-out **hard slice** for honest stress-testing.

## 5. Evaluation

### Metrics
- **Sentence-level exact-match accuracy** (headline metric).
- **Macro per-semiotic-class accuracy** (exposes rare-class failures).
- **Per-class accuracy.**
- **Unrecoverable / "silly" error rate** (per Sproat & Jaitly) — semantically catastrophic mis-reads. The model is tuned to minimize these even at the cost of some benign errors.

### Baseline comparison

The baseline to beat is a **context-blind rule normalizer** (`src/audiobook_ai/models/baseline_rules.py`): ordered regex spans + an abbreviation dictionary + `num2words` / pure-Python expanders. It is strong on easy cases but fails on ambiguous ones.

**Validated baseline numbers on the synthetic test distribution** (sentence exact-match, EM):

| Slice | Baseline EM |
|---|---|
| Easy | 0.945 |
| Hard | 0.006 |
| Overall | 0.712 |

The context-aware trained model is expected to reach high accuracy on **both** slices, beating the baseline **overall** and **especially on the hard slice** — where the context-blind baseline collapses to **0.006**. This is reported as an honest, reproducible result.

## 6. Limitations and Biases

- **Synthetic-to-real gap.** The model is trained primarily on a synthetic corpus. Real-world prose (idiosyncratic formatting, OCR noise from scanned PDFs, domain jargon) may differ from the training distribution.
- **English / US-centric.** Normalization conventions (date order, currency, address abbreviations) reflect English and US-centric usage. Behavior on other locales is not guaranteed.
- **Possible unrecoverable errors.** Despite minimization, the model can still produce semantically catastrophic mis-reads on rare or adversarial inputs. The pipeline mitigates this with a rule guardrail, a confidence-based escalation gate, and unrecoverable-error counting — but the model alone offers no hard guarantee.
- **Class imbalance.** `PLAIN` is capped at ~30% during training; rare classes still see fewer examples, and macro-class accuracy is reported specifically to surface those weak spots.

---

## 7. Ethical Considerations

- **Confidence-aware deployment.** In the pipeline, each segment carries a length-normalized sequence-probability confidence. Below the threshold (`0.55`), the output is flagged and may be escalated to an optional LLM brain that validates and falls back to deterministic rules; neural-vs-baseline disagreements are recorded.
- **Hallucinated pronunciations** are mitigated by the rule guardrail, the downstream audio-QA re-synthesis gate, and explicit unrecoverable-error counting.
- **PII in books** (phone numbers, addresses, emails, names): logging is minimized, with TTL cleanup; NER is used for attribution only.
- **Copyright.** Converting a book is a derivative work — rights / public-domain status must be verified, and the input's SHA-256 is recorded for provenance.
- **License hygiene.** The model and its base are Apache-2.0, keeping it eligible for the project's commercial path (MIT/Apache-only). The model performs no voice cloning; voice generation lives entirely in the separate, consent-gated TTS backend.

## 8. How to Use

Load the model with `transformers` and prefix every input with `normalize: `. Greedy or beam decoding both work; the pipeline uses sequence probability as a confidence signal.

```python
from transformers import AutoTokenizer, AutoModelForSeq2SeqLM

model_id = "audiobook-ai-tn-byt5-small"  # local path or Hub id
tokenizer = AutoTokenizer.from_pretrained(model_id)
model = AutoModelForSeq2SeqLM.from_pretrained(model_id)

PREFIX = "normalize: "

def normalize(text: str) -> str:
    inputs = tokenizer(PREFIX + text, return_tensors="pt")
    output_ids = model.generate(**inputs, max_new_tokens=256)
    return tokenizer.decode(output_ids[0], skip_special_tokens=True)

# Examples spanning several semiotic classes
print(normalize("The deal closed at $5.2M in 1984."))
print(normalize("See Dr. Smith on St. Paul St. at 9:45 AM."))
print(normalize("Read Chapter IV, then mix 3/4 cup."))
```

### Integration notes
- Call the model **per segment**, after sentence segmentation (the pipeline uses `pysbd` + TTS-aware chunking) and before synthesis (the default backend is `microsoft/speecht5_tts`).
- Use the length-normalized sequence probability as a **confidence score**; route low-confidence segments to the rule normalizer or the optional LLM brain.
- For small/CPU-only environments, train and load the `t5-small` fallback variant via the same `AutoModelForSeq2SeqLM` interface and the same `normalize: ` prefix.

## Dataset Summary

This is a **reproducible synthetic corpus** of `(written, spoken)` pairs for **Text Normalization (TN)** — the written-to-spoken conversion that a TTS system must perform before synthesis. It exists because a naive TTS reads strings like `$5.2M`, `1984`, `Dr.`, `Chapter IV`, `3/4`, and `9:45 AM` incorrectly; converting these to their spoken form is the hard, NLP-heavy part of audiobook production.

The corpus is generated by a single deterministic generator script (`src/audiobook_ai/data/tn_corpus.py`) rather than downloaded, because **there is no permissively-licensed English TN-Challenge mirror on the Hugging Face Hub** (the canonical ids `google/text_normalization` and `cestwc/text-normalization` both 404). The generator produces **context-rich** pairs across **16 semiotic classes**, with deliberately **injected ambiguity** so that a context-aware model can be shown to beat a context-blind rule baseline.

The intended consumer is a Seq2Seq normalization model — primarily `google/byt5-small` (byte/char-level, robust to numbers/symbols/OOV), with `google-t5/t5-small` as a small-GPU fallback — trained with the task prefix `"normalize: "`.

## Semiotic Class Schema

Every pair belongs to one of **16 semiotic classes**. The schema follows the standard TN taxonomy used in the Sproat & Jaitly text-normalization literature. The table below names each class and gives an illustrative `(written → spoken)` example.

| # | Class | Description | Example: `written` → `spoken` |
|----|-------------|----------------------------------------------|--------------------------------|
| 1 | `PLAIN` | Ordinary words passed through unchanged | `the` → `the` |
| 2 | `PUNCT` | Punctuation (spoken or silent as appropriate) | `,` → `,` |
| 3 | `CARDINAL` | Counting numbers | `1984 people` → `one thousand nine hundred eighty-four people` |
| 4 | `ORDINAL` | Rank / position numbers | `21st` → `twenty-first` |
| 5 | `DECIMAL` | Numbers with a fractional point | `3.14` → `three point one four` |
| 6 | `DIGIT` | Numbers read digit-by-digit | `007` → `zero zero seven` |
| 7 | `MONEY` | Currency amounts | `$5.2M` → `five point two million dollars` |
| 8 | `MEASURE` | Quantity + unit | `5 km` → `five kilometers` |
| 9 | `DATE` | Calendar dates (day often as ordinal) | `1984-03-02` → `March second, nineteen eighty-four` |
| 10 | `TIME` | Clock times | `9:45 AM` → `nine forty-five A M` |
| 11 | `FRACTION` | Fractions | `3/4` → `three quarters` |
| 12 | `TELEPHONE` | Phone numbers (digit grouping) | `555-0142` → `five five five, zero one four two` |
| 13 | `ELECTRONIC` | URLs, emails, handles | `a@b.com` → `a at b dot com` |
| 14 | `ABBREV` | Abbreviations & contextual titles | `Dr.` → `Doctor` |
| 15 | `ROMAN` | Roman numerals in context | `Chapter IV` → `Chapter Four` |
| 16 | `VERBATIM` | Read letter-by-letter / as-is | `NASA` → `N A S A` |

> **Note:** Example spoken forms above illustrate the target convention; exact surface strings are produced deterministically by the generator and its expanders (`num2words` plus pure-Python expanders in `src/audiobook_ai/models/expanders.py`).

### Example pairs per class (with injected ambiguity)

The hard cases — where context decides the reading — are what make this corpus useful. Examples:

| Class | Ambiguous `written` | Context-dependent `spoken` |
|-------------|------------------------------|-----------------------------|
| `CARDINAL` vs `DATE` | `1984 people` | `one thousand nine hundred eighty-four people` (count) |
| `DATE` | `in 1984` | `in nineteen eighty-four` (year) |
| `ABBREV` | `St. Louis` | `Saint Louis` (title) |
| `ABBREV` | `123 Main St.` | `123 Main Street` (street) |
| `ROMAN` | `Chapter IV` | `Chapter Four` (numeral) |
| `DATE` | `March 2nd` | `March second` (day-as-ordinal) |

A context-**blind** rule normalizer cannot disambiguate `St.` → `Street`/`Saint` or `1984` → count/year reliably; a context-**aware** Seq2Seq model can. This contrast is the experimental core of the project.

## Splits and Sizes

Default generation sizes (configurable via the generator):

| Split | Size | Purpose |
|--------|--------|----------------------------------------------------------|
| `train` | 60,000 | Model training |
| `val` | 4,000 | Validation / early-stopping (on `sentence_accuracy`) |
| `test` | 4,000 | Held-out evaluation (mixed easy + hard distribution) |
| `hard` | 1,500 | Dedicated hard slice of ambiguous, context-sensitive cases |

**Leakage-free splits:** no `val`/`test` input appears in `train`. Pairs are **deduplicated before splitting**, and `PLAIN` is **capped at ~30%** of the corpus to prevent the model trivially overfitting to pass-through tokens.

## Generation Process

The corpus is produced by `src/audiobook_ai/data/tn_corpus.py`, a deterministic generator that:

1. **Samples** templates per semiotic class, embedding the target token in realistic sentence context (so the model sees surrounding words, not isolated tokens).
2. **Expands** numbers, money, measures, dates, times, fractions, etc. into spoken form using `num2words` and pure-Python expanders.
3. **Injects ambiguity** so identical or near-identical surface forms map to different spoken outputs depending on context — e.g. `St.` (Street vs Saint), `1984` (count vs year), dates with day-as-ordinal, and Roman-numeral contexts that the rule baseline mishandles.
4. **Deduplicates** pairs, **caps** the `PLAIN` proportion, then performs a **leakage-free split** into `train` / `val` / `test` / `hard`.

Because generation is code-driven and deterministic, the entire corpus is **fully reproducible** from the script — no external dataset download is required.

## Intended Use

- **Primary:** Train and evaluate a Seq2Seq Text-Normalization model (`google/byt5-small`; fallback `google-t5/t5-small`) for the audiobook pipeline's normalization stage, using the `"normalize: "` task prefix.
- **Benchmarking:** Compare a trained context-aware model against the **context-blind rule baseline** (`src/audiobook_ai/models/baseline_rules.py`), which combines ordered regex spans, an abbreviation dictionary, and `num2words`/pure-Python expanders.
- **Evaluation metrics it supports:**
  - **Sentence-level exact-match accuracy** (headline metric).
  - **Macro per-semiotic-class accuracy** (exposes rare-class failures).
  - **Per-class accuracy.**
  - **Unrecoverable / "silly" error rate** (Sproat & Jaitly) — semantically catastrophic mis-reads to minimize, even at the cost of some benign errors.

### Reference baseline results (synthetic test distribution)

The context-blind baseline scores (sentence exact-match, EM):

| Slice | Baseline EM |
|----------|-------------|
| easy | 0.945 |
| hard | 0.006 |
| overall | 0.712 |

The trained context-aware model is expected to reach high accuracy on **both** slices — beating the baseline overall and especially on the **hard** slice. This is an honest, reproducible target, not a claimed headline number.

## Out-of-Scope Use

- **Not a general-purpose NLP corpus.** It targets TTS-oriented written→spoken normalization, not translation, summarization, or comprehension.
- **Not multilingual.** English only; do not expect correct normalization of other languages.
- **Not a benchmark of real-world distribution.** It is **synthetically generated**, so token/class frequencies reflect the generator's templates and the injected-ambiguity design, not naturally occurring text statistics.
- **Not a substitute for a licensed TN-Challenge corpus.** It was created precisely because no permissive English TN-Challenge mirror exists on the Hub; results should not be reported as if measured on the official challenge data.

## Known Biases and Limitations

- **Synthetic distribution bias.** Class balance and phrasing come from generator templates; rare or idiosyncratic real-world forms may be under-represented or absent.
- **Convention lock-in.** Spoken targets follow the generator's chosen reading conventions (e.g. how times, telephone groupings, or decimals are verbalized). A model trained here inherits those conventions, which may not match every audiobook house style.
- **Ambiguity coverage is curated, not exhaustive.** Injected ambiguities (`St.`, `1984`, day-as-ordinal dates, Roman contexts) are representative but cannot cover every context-dependent case in natural prose.
- **English-only.** No cross-lingual generalization.
- **PLAIN cap is a design choice.** Capping `PLAIN` at ~30% improves training signal on hard classes but makes the corpus' class mix deliberately unlike raw text, where plain words dominate.
- **Evaluation is on synthetic data.** Strong numbers here do not guarantee identical performance on arbitrary real books; the optional sanity set and the audiobook pipeline's runtime guardrails exist partly to mitigate this.

---

## Ethical Considerations

- **Downstream PII.** Real books fed to the pipeline can contain phone numbers, addresses, emails, and names; the project minimizes logging and applies TTL cleanup. The corpus itself uses synthetic, non-personal example tokens (e.g. fictitious phone/email patterns).
- **Copyright.** This corpus is generated text and is itself unencumbered, but the broader pipeline treats converting a book as a derivative work — rights/public-domain status must be verified and input `sha256` recorded.

## Maintenance

- **Versioning:** Corpus changes are tracked via the generator code and the project's model registry conventions (`tn_meta.json` + a `latest` pointer, with `repo@revision` pins for any external ids).
